Interactive Storyboard Setup

App Setup

The application is built on node version 12.16.1 using React version 16.13.1. To run the application, one needs to have node on his/her computer.

Running the application for the first time

While running the application for the first time, one will need to install various dependencies. This can be done by following steps:

  • conda create -n dss-dev python=3.6
  • conda activate dss-dev (or) activate dss-dev
  • pip install -r requirements.txt --trusted-host=dss-new.pfizer.com:10000
  • conda install -c conda-forge nodejs=15.10.0
  • npm install

Running the application after first time

  • conda activate dss-dev && app.bat (or) activate dss-dev && app.bat

API Flow

Uploading the app on DSS

To upload the application on the DSS, we first need to build the application in a way that PUBLIC_URL points to the root folder of the plugin on the DSS. Currently, app is configure for the purpose - 

  • Build the application by running "npm run build:dss" command
  • Go to DSS, navigate to "Plugins", search for "Interactive Storyboard UI"
  • Delete the "my-app" folder from resources subdirectory
  • Create following directory:
    • my-app > build > static > css, js, media
  • Upload all the files in the directory in similar way as generated in the build
  • Copy the content from "index.html" and paste into webapps > form-ui-and-story-generator > body.html file
  • Save and refresh the application


Key points related to DSS webapps written in React

  • For security reasons, DSS does not allow the webapps to directly access backend APIs.
  • DSS adds some utility functions on top of our application and web apps are supposed to get the API access through this utility layer
  • The utility layer is added only after DOM is fully loaded. This creates a problem for our application because our app needs data from the backend to generate DOM
  • To tackle this, we load an invisible button at first, and click the button through code and make api calls on the button click

Environment Variables

  • REACT_APP_MODE - If the variable is set to "dev", the application runs in the development mode (on local machine)
  • PUBLIC_URL - To make the app DSS friendly, the value is set to "../../resource/my-app/build/" so that the body.html file points to root folder of the plugin on the dss

Other important point

  • Sometimes app throws error on "npm install" when it should not. In such cases, following steps should be tried:
    • Try deleting package-lock.json
    • Try replacing package-lock.json from previous version
  • In some cases, the app might throw an error while running it. In such cases, try setting SKIP_PREFLIGHT_CHECK=true


Additional Comments

Charting library being used in the two applications, ie, Storyboard UI & Infographic UI is same except-

  • ShowCustomTooltip & MoveCustomTooltip in mouseover & mousemove events have one additional parameter of chartContainer
  • ShowCustomTooltip & MoveCustomTooltip in chartUtils differ in how they get x & t coordinates of the cursor. 
  • shouldComponentUpdate method is used in Infographic UI

To make charting library exactly same, passing additional parameters in storyboard ui is recommended
